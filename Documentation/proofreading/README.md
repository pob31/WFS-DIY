# Translation proofreading

This directory contains one checklist per non-English locale. Each file lists every translation key referenced by `LOC()` in the source code, paired with the English source value and the current translated value, with an inline checkbox for sign-off.

Each file covers ~1520 keys. They were autogenerated from `Resources/lang/en.json` and the locale's `Resources/lang/<locale>.json`.

## How to use

1. Pick a locale file matching your fluency.
2. Walk through it section by section. Sections are grouped by top-level namespace (`ai.history`, `inputs.help`, `network.dialogs`, etc.) so you can stop mid-file and resume without losing your place.
3. For each entry:
   - **Translation reads naturally and matches the meaning** → mark `[x] OK` (or leave blank if you trust the silent default).
   - **Translation is wrong, awkward, or has a typo** → write the corrected text after `Fix:`.
   - **Translation is missing** → entry will say `(missing — falls back to key string in UI)`. Write the correct translation after `Fix:`.
   - **Translation is identical to the English source** → may be intentional (loanword, proper noun, technical term, short token that's the same in both languages). Mark `[x] OK` to confirm intent, or write `Fix:` if it should actually be translated.

## What counts as "intentionally identical to English"

These categories are deliberately not translated:
- **Audio loanwords**: Sampler, Tracking, Pre, Post, ON/OFF, LFO, HF, EQ, AI, OSC, MQTT
- **Proper nouns**: QLab, Lightpad, Stream Deck, ADM-OSC, PosiStageNet, RTTrP, Ableton Live, Roli, Xencelabs Quick Keys
- **Technical terms**: OSC Path:, Localhost, http://localhost:..., file extensions
- **Math notation**: r θ φ, r θ Z, single-letter axis labels (X, Y, Z, Q)
- **Short identical short tokens**: e.g. German "Position" == English "Position", Spanish "Error" == English "Error"
- **Status codes**: ON, OFF, OK, single dB values

If you see one of these flagged as identical, marking `[x] OK` is the right answer.

## Special characters to preserve

- **Curly braces** `{name}`, `{num}`, `{path}`, `{cluster}`, `{verb}`, `{description}`, `{message}`, `{from}`, `{to}`, `{n}`, `{id}`, `{current}`, `{existing}` etc. — these are runtime placeholders. They get replaced with actual values at display time. **Keep them in the translation, in the same position relative to the surrounding text.**
- **`\n`** — literal newline in the rendered UI. **Preserve it.** Used in multi-line tooltips and dialogs.
- **`\"`** — escaped double quote. Preserve.

## Workflow

Two reasonable workflows:

**A. Native speaker reviews independently.**
- Open the file for your locale in any markdown viewer.
- Walk through and annotate `Fix:` lines.
- Send the annotated file back; a maintainer will apply the fixes to `Resources/lang/<locale>.json`.

**B. Side-by-side review with the developer.**
- Open the locale's `.md` file and `Resources/lang/<locale>.json` simultaneously.
- For each entry the proofreader flags, edit the JSON file directly.
- Mark the checklist entry `[x] OK` once applied.

## After proofreading

Once a locale is fully reviewed:
1. Apply all `Fix:` edits to `Resources/lang/<locale>.json`.
2. Validate the JSON syntax (any JSON parser will do).
3. Rebuild the application and spot-check a few corrected strings in the UI.
4. The checklist file can be archived (e.g. moved to `Documentation/proofreading/done/`) or kept as the source-of-truth audit log.

## Regenerating the checklists

The files were generated by walking the JSON and source code at a snapshot in time. If new `LOC()` keys are added or removed, the checklists will drift. To regenerate, the script logic is:

1. Grep all `LOC("…")` calls in `Source/` for direct keys.
2. Grep all dotted-path string literals matching keys in `en.json` for indirect keys (e.g. struct fields passed to `LOC(var)`).
3. For each key, emit the English source from `en.json` and the locale's translation from `<locale>.json`.
4. Group by `<namespace1>.<namespace2>` and sort alphabetically.

A maintainer with PowerShell access can rerun the generation any time.
